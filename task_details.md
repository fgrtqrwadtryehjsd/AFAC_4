# AFAC2026 赛题四：金融长文本Agent的动态记忆压缩与高效问答挑战

> 来源：https://tianchi.aliyun.com/competition/entrance/532486

## 一、任务介绍

金融行业存在大量专业性强、篇幅长、结构复杂的文本资料，例如保险条款、监管法规、债券募集说明书、上市公司年报和行业研究报告。这类文档通常包含精确的期限、比例、例外条件、权利义务关系和跨章节引用，单纯依赖通用知识或关键词匹配难以稳定回答问题。

随着大语言模型和检索增强生成技术的发展，金融场景对模型的要求已经从“能否阅读单段文本”扩展到“能否在多份长文档之间定位证据、比较差异、完成计算和判断”。模型不仅需要准确检索相关片段，还需要理解金融语义、识别干扰项，并在答案格式上保持严格可评测。

然而，在实际应用中，金融长文本的篇幅往往远超模型上下文窗口的经济使用区间。直接将全文输入模型会导致 Token 消耗量巨大，推理成本难以控制；简单截断或朴素摘要又容易丢失关键事实、业务逻辑和约束条件，造成事实性错误或推理链断裂。

本赛题面向金融长文档问答任务，要求参赛者在不修改基座模型参数、仅通过Qwen系列模型 API 调用的前提下，设计 Agent 层面的记忆流转、动态压缩和上下文优化策略。赛题围绕五类典型金融文本构建评测集，考查系统在多文档阅读、事实核验、复杂推理、跨文本比较、答案规范化和 Token 成本控制方面的综合能力。要求参赛者：

- **不修改基座模型参数**
- **仅通过 Qwen 系列模型 API 调用**
- 设计 Agent 层面的**记忆流转、动态压缩和上下文优化策略**

赛题围绕五类典型金融文本构建评测集，考查：
- 多文档阅读
- 事实核验
- 复杂推理
- 跨文本比较
- 答案规范化
- Token 成本控制

## 二、任务目标

构建金融长文本 Agent，在尽可能少的 Token 消耗下准确回答问题。

### 题目类型
| 类型 | 格式 | 说明 |
|------|------|------|
| 单选题 | mcq | 从 A/B/C/D 选唯一正确答案 |
| 多选题 | multi | 从 A/B/C/D 选所有正确答案，按字母序排列如 ABC |
| 判断题 | tf | 使用 A/B 表示正确或错误 |

### 模型限制
- **推理问答阶段**：严格使用 Qwen 系列模型
- **仅限**通过阿里云百炼平台或魔搭社区调用 Qwen 系列 API
- **不得使用**任何其他开源或闭源模型替代推理

### 考查能力
1. 金融长文档阅读理解
2. 多文档检索与定位
3. 跨文档比较分析
4. 数值计算与条件推理
5. 动态记忆压缩
6. Token 成本优化
7. 抗干扰能力

### 可用技术方向
动态摘要抽取、Prompt Engineering、思维链提示、Agent 记忆管理、上下文窗口优化、RAG、结构化信息抽取、答案后处理

## 五、评测指标

### 计算公式

```
Accuracy = Correct / Total
TokenBudget = 5,000,000
TokenScore = max(0, min(1, (TokenBudget - TotalTokens) / TokenBudget))
FinalScore = 100 * Accuracy * (0.7 + 0.3 * TokenScore)
```

- 准确率决定主体得分（70%权重）
- Token 效率最多影响 30% 加权系数
- 未提交有效 Token 统计或 TotalTokens <= 0 则 TokenScore = 0

### 同分排序规则
1. 准确率更高者靠前
2. TotalTokens 更低者靠前
3. 提交时间更早者靠前

### 答案标准化
- 单选/判断：以首个有效答案字母为准
- 多选：去重排序后完全匹配，无部分分

### 基准模型
评测基准模型为 **Qwen3.6-plus**

**优化方向**：

3. 证据聚合（同题多文档、多段落联合判断）

5. 领域知识增强（保险/监管/合同/财报/研报分别设计提示词和后处理规则）

## 八、推荐解题流程

1. **文档预处理**：抽取 PDF 文本，保留标题、页码、章节、表格和条款编号
2. **文档组织与索引**：按领域构建关键词索引和结构化字段索引
3. **题目解析**：识别题型、关键词、实体、时间、金额、指标和候选文档
4. **证据检索**：根据题目和选项分别检索证据，必要时多轮查询
5. **推理作答**：让模型基于证据逐项判断选项真假
6. **答案校验**：根据题型约束输出合法答案字母
7. **结果汇总**：生成符合提交格式的 csv/json 文件
