# AFAC2026 赛题四 实验记录

## 赛题信息

- **赛题**：金融长文本Agent的动态记忆压缩与高效问答挑战
- **评分公式**：`FinalScore = 100 × Accuracy × (0.7 + 0.3 × TokenScore)`
- **TokenScore**：`max(0, min(1, (5,000,000 - TotalTokens) / 5,000,000))`
- **Token 预算**：5,000,000
- **模型限制**：仅 Qwen 系列 API
- **A 榜**：100 题，已知 doc_ids
- **B 榜**：100 题，未知 doc_ids
- **当前最高分**：85

- 多选题 ABCD 全选几乎不可能正确，需要硬性限制

### V6 — Query-Focused Evidence Extraction (QFEE) 🔥

| 指标 | 值 |
|------|-----|
| 日期 | 2026-06-15 |
| 推理模型 | qwen-plus |
| 改进 | 根本性架构改变：**跳过全局压缩**，每题单独提取证据 |
| 有效答案 | 100/100 |
| 推理 Token | 1,255,665 |
| 压缩 Token | 0（无压缩） |
| 总 Token | 1,255,665 |
| TokenScore | 0.7489 |
| **A 榜分数** | **25.8905** |
| 状态 | ✅ 运行完成，答案模式明显改善 |

**V6 核心创新（受论文启发）**：
1. **FinCARDS**: Query-Focused Evidence Extraction — 每题提取关键词+BM25双路召回，不再做全局文档压缩
2. **Acon**: RRF 融合多路召回 — BM25(1.0权重) + 关键词精确匹配(1.5权重)，关键词在金融领域更可靠
3. **Self-RAG/Self-Critique**: 多选题只允许删除不允许添加
4. **答案多样性**: multi题从ABCD全选变为 AB/ABC/B/BD/ACD 等多样答案

**答案分布变化对比**：

| 类型 | V5 (压缩) | V6 (QFEE) |
|------|-----------|-----------|
| multi-ABCD全选 | 35题 | 0题 ✅ |
| multi-ABC | 1题 | 30题 |
| multi-AB | 3题 | 14题 |
| multi-单选 | 2题 | 12题 |
| mcq全选A | 12题 | 分散到A/B/C/D |
| tf | A/B基本各半 | A/B基本各半 |

**关键教训**：
- ❌ 全局压缩 = 信息丢失 = 准确率崩塌（V5: 20.27分）
- ✅ Query-Focused Evidence Extraction = 无损提取 = 更高准确率
- ✅ 关键词精确匹配在金融术语领域比纯BM25更可靠
- ✅ 多选题硬限制+保守Self-Critique大幅减少过度选择

### V7 — 渐进式记忆压缩 Agent (Progressive Memory Compression) 🔥🔥

| 指标 | 值 |
|------|-----|
| 日期 | 2026-06-15 |
| 推理模型 | qwen-plus |
| 改进 | **文档级结构化Card(规则提取,0 Token) + Card缓存复用 + 条款精准定位 + 自适应证据量** |
| 有效答案 | 100/100 |
| 总 Token | 2,333,129 |
| TokenScore | 0.5334 |
| **A 榜分数** | **未单独提交**（合并入V7.1） |
| Card 构建耗时 | 2.3s (零Token) |
| Card 缓存 | 68个文档, 231次访问 |

**V7 核心技术创新**：
1. **DocumentCard**: 基于规则的结构化索引（条款/实体/指标/术语），零Token成本
2. **AgentMemory**: 渐进式记忆层，文档Card首次构建后跨题目复用
3. **Card引导检索**: Card不是答案，是索引——匹配命中的实体/术语指导关键词检索
4. **条款精准定位**: 当题目引用条款号时，直接从全文提取条款原文
5. **自适应证据量**: multi=30K, tf=12K, mcq=20K

**答案分布**：

| 领域 | V7 分布 |
|------|---------|
| financial_contracts | A(6) ABC(4) B(3) AD(2) |
| financial_reports | A(9) AB(4) B(3) D(2) |
| insurance | A(7) ABC(5) B(2) AB(2) C(2) |
| regulatory | A(8) ABC(3) B(3) ABD(3) |
| research | A(8) ABC(4) AB(3) B(2) |

**问题分析**：
- 条款引用0次命中（题目不直接引用条款号）
- Card实体匹配49题/术语匹配55题有效
- 证据量过大(平均32K,最大138K)导致Token膨胀

### V8 — 精确证据推理 Agent (选项去偏+全文证据+精确推理) 🔥🔥🔥

| 指标 | 值 |
|------|-----|
| 日期 | 2026-06-15 |
| 推理模型 | qwen-plus |
| 改进 | **选项去偏(Shuffle Options) + 40K全文证据 + 精确推理Prompt** |
| 有效答案 | 100/100 |
| 总 Token | 2,721,971 |
| TokenScore | 0.4556 |
| **A 榜分数** | **25.10** ❌ |

**V8 核心突破**：
1. **选项去偏**: 打乱ABCD顺序消除模型A-bias（V7.1: 51/69单选选A→V8更均匀）

**❌ 失败分析**：
- 选项打偏反而降低准确率（30.28→25.10）
- 可能原因1：金融题选项有逻辑顺序，打乱破坏了推理
- 可能原因2：A-bias不是真正偏差——A确实更常是正确答案
- 可能原因3：模型在熟悉格式(A/B/C/D)下推理更好
- TokenScore从0.635降至0.456（40K证据量太大）

**关键教训**：选项打偏不可行，恢复V7.1架构
